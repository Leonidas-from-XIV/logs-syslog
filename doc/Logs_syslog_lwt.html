Module Logs_syslog_lwt

module Logs_syslog_lwt: sig .. end
Logs reporter via syslog, using Lwt

Please read Logs_syslog first.

val udp_reporter : ?hostname:string ->
       Lwt_unix.inet_addr -> ?port:int -> unit -> Logs.reporter Lwt.t
udp_reporter ~hostname remote_ip ~port () is reporter, which sends syslog message using the given hostname to remote_ip, remote_port via UDP. The hostname default to Lwt_unix.gethostname (), port to 514.
val tcp_reporter : ?hostname:string ->
       Lwt_unix.inet_addr ->
       ?port:int -> unit -> (Logs.reporter, string) Result.result Lwt.t
tcp_reporter ~hostname remote_ip ~port () is Ok reporter or Error msg. The reporter sends each log message to remote_ip, port via TCP. If the initial TCP connection to the remote_ip fails, an Error msg is returned instead. If the TCP connection fails, the log message is reported to standard error, and attempts are made to re-establish the TCP connection. Each syslog message is terminated with a 0 byte. The hostname default to Lwt_unix.gethostname (), port to 514.
